# 喵喵猜词重连 — 端到端手动测试清单

## 测试准备

| 条件 | 说明 |
|------|------|
| 两个浏览器窗口 | Chrome/Edge 同设备用于基础测试 |
| 两台设备不同网络 | WiFi + 4G/5G 用于 TURN 测试 |
| 至少 3 名参与者 | 测试 migration 和 auto-skip |
| 浏览器 DevTools | 监控控制台日志和 ICE 连接状态 |

---

## 测试场景

### 1. 房主 WiFi 断开 + 自动重连
| 步骤 | 预期 |
|------|------|
| 房主创建房间，2 位访客加入，游戏开始 | |
| 房主：关闭 WiFi，等待 10 秒，重新打开 | |
| 观察访客画面 | 出现 "🔄 正在重连..." 横幅，短暂 overlay（如果超过 3s） |
| WiFi 恢复后 | 1-2 秒内自动重连成功，游戏继续同一阶段 |
| 检查访客连接状态 | 访客之间的连接不受影响 ✅ |
| **通过** | ☐ |

### 2. 访客 WiFi 断开 + 自动重连
| 步骤 | 预期 |
|------|------|
| 1 位访客，游戏中 | |
| 访客：关闭 WiFi，等待 10 秒，重新打开 | |
| 观察访客画面 | 出现 overlay（含尝试次数），弹跳圆点动画 |
| Wi-Fi 恢复后 | 自动重连成功，全量状态恢复（手牌、阶段、分数） |
| overlay 消失 | ✅ |
| **通过** | ☐ |

### 3. 房主关闭浏览器标签页（host migration）
| 步骤 | 预期 |
|------|------|

| 等待约 10 秒 | 心跳检测到死连接 |

| 游戏继续 | 与新主机正常进行 |
| 访客的 overlay | 不应闪烁 ✅ |
| **通过** | ☐ |

### 4. 两位访客同时断开 + 独立重连
| 步骤 | 预期 |
|------|------|
| 两位访客同时断开 WiFi | |
| 各自重连 | 各自独立自动重连，不互相干扰 |
| 状态恢复 | 两位都恢复全量状态 |

### 8. 所有访客断开
| 步骤 | 预期 |
|------|------|
| 所有访客断开 WiFi | |
| 房主画面 | 所有玩家显示为离线 |
| 游戏暂停（不结束） | |
| 访客在 60 秒内重连 | 恢复，游戏继续 |
| 超过 60 秒 | 离线玩家会被从房间移除 |
| 如果 < 3 人在线 | 游戏结束 ✅ |
| **通过** | ☐ |

### 9. ICE 状态瞬态抖动（disconnected → connected）
| 步骤 | 预期 |
|------|------|

| 游戏 | 不受干扰继续 |
| **通过** | ☐ |

### 10. ICE 进入 failed 状态
| 步骤 | 预期 |
|------|------|
| 通过 DevTools 或网络中断触发 `iceConnectionState: 'failed'` | |
| 观察 | ICE 立即重启 |
| 自动重连启动 | |
| 如果恢复 | 游戏继续 |
| 如果达到最大尝试次数 | 「连接失败」overlay + 手动重连按钮 ✅ |
| **通过** | ☐ |

### 11. 快速断开/重连（网络抖动）
| 步骤 | 预期 |
|------|------|

| **通过** | ☐ |
