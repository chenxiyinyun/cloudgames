# 2026-05-25 截码战修复记录

## 修复内容

### 1. 核心流程修复 (gameEngine.js)
- **修正轮换顺序**: 实现正确的4人轮换机制 (黑A → 白A → 黑B → 白B → 循环)
- **新增队内投票系统**: 当队友两人猜测不一致时，需要投票统一决定
- **修正角色分配**: 每回合只有一个情报官，不再同时设置多个
- **新增游戏阶段**: `TEAM_VOTING` 阶段用于队内投票
- **重构猜测提交**: 分离为 `submitTeamGuess` (队友) 和 `submitOpponentGuess` (对方)

### 2. 状态管理更新 (gameStore.js)
- 新增消息类型: `SUBMIT_TEAM_GUESS`, `SUBMIT_OPPONENT_GUESS`, `SUBMIT_TEAM_VOTE`
- 更新状态同步逻辑，支持新的 `teamVotes` 数据结构
- 新增处理函数: `handleSubmitTeamGuess`, `handleSubmitOpponentGuess`, `handleSubmitTeamVote`

### 3. UI界面优化 - 二战电报风格
- **配色方案**: 纸张米色背景、深褐色文字、复古印章红色
- **字体**: Courier Prime (打字机) + Noto Serif SC (宋体)
- **视觉元素**: 电报头部 (TOP SECRET + 摩斯电码)、军牌风格队伍信息、印章效果标记、文件夹风格面板、纸张纹理和横线背景
- **术语更新**: "房间"→"任务", "玩家"→"特工", "情报官"等

### 4. 断线重连与状态恢复
- **stateCache.js**: 使用 localStorage 持久化游戏状态，30分钟自动过期
- **刷新恢复弹窗**: 显示断线恢复对话框，展示任务编号、特工代号、身份

### 5. 游戏规则修正
- **敌方关键词可见**: 符合原版 Decrypto 规则，双方都能看到所有关键词
- **token字段统一**: `interceptTokens`→`interceptionTokens`, `missTokens`→`miscommunicationTokens`
- **密码允许重复**: generateCode() 改为随机选取（允许如1-1-3）
- **投票UX改进**: 投票面板增加"自定义密码"选项
- **拦截方对称化**: 拦截方两人各自提交，不一致则投票

### 6. 后续架构迁移

以上修复最初基于早期联机架构实现。项目随后整体迁移到**服务器权威 WebSocket** 架构：服务器持有权威状态，客户端精简为瘦客户端（只发意图、收权威状态），详见 `src/shared/ws/` 共享网络层。
