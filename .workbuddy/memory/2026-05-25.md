## 访客托管机制 (第三次提交)

### 1. 网状 P2P 拓扑 (p2p.js)
- **访客互连**: 所有访客之间建立直接 PeerJS 连接，不再只依赖房主
- **新增 connectToPeer()**: 主动连接到指定 peerId
- **访客监听连接**: 访客也监听 `peer.on('connection')` 接受其他访客直连

### 2. 房主迁移逻辑 (gameStore.js)
- **断线检测**: 访客检测到房主断开时触发 `handleHostDisconnect()`
- **自动选举**: 按加入顺序（order）选举最小 order 的在线玩家为新房主
- **状态接管**: 新房主获得 `cachedRoom` 控制权，更新 `hostId`
- **广播通知**: 发送 `HOST_MIGRATION` 消息给所有 peers
- **连接切换**: 其他访客自动连接到新房主的 peer

### 3. 消息类型新增
- `PEER_LIST`: 房主发送已连接 peer 列表给新加入者
- `CONNECT_TO_PEER`: 要求某 peer 连接到指定 peer
- `HOST_MIGRATION`: 通知所有 peer 房主变更

### 4. 文件变更
- `src/services/p2p.js` - 重写为网状拓扑，新增 connectToPeer
- `src/stores/gameStore.js` - 新增房主迁移、peer 互连逻辑
